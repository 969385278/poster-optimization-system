# 基于眼动与表情反馈的海报自优化系统

## 项目简介

本项目面向海报阅读与传播场景，设计并实现了一个“生成 - 观察 - 诊断 - 优化”的闭环系统。
系统结合海报生成、摄像头采集、面部特征分析、注视点估计、语义区域统计与反馈驱动优化机制，对用户的阅读行为进行分析，并据此生成海报优化建议或优化版本。

项目重点不在于单次生成“好看”的海报，而在于通过真实阅读反馈提升海报的信息传达效率，使标题、主视觉、时间地点、二维码等关键内容能够被更稳定地注意和理解。

## 主要功能

- 海报生成与展示
- 默认模板海报加载
- 本地海报图片上传
- 摄像头读取与本地采集
- 人脸与眼动特征跟踪
- 注视点估计与五点校准
- 海报语义区域（AOI）命中统计
- 阅读反馈分析与润色
- 海报优化建议生成
- 优化版本海报输出

## 运行方式

### 1. 安装依赖

```bash
npm install
```

### 2. 配置环境变量

复制示例配置文件：

```bash
copy .env.example .env
```

然后根据实际情况在 `.env` 中填写相关配置，例如：

- `ARK_API_KEY`
- `ARK_TEXT_MODEL`
- `ARK_IMAGE_MODEL`
- `ARK_VISION_MODEL`
- `DEEPSEEK_API_KEY`
- `DEEPSEEK_TEXT_MODEL`

如果未配置外部模型接口，系统会使用部分本地回退逻辑运行演示流程。

### 3. 启动前端

```bash
npm run dev:client
```

### 4. 启动后端接口

```bash
npm run dev:api
```

### 5. 或直接启动开发模式

```bash
npm run dev
```

## 核心流程

1. 输入海报需求或加载默认模板
2. 生成海报初稿或读取已有海报
3. 打开摄像头，进行五点校准
4. 用户阅读海报，系统记录注视点与表情特征
5. 对标题、主视觉、时间地点、二维码等语义区域进行统计分析
6. 输出阅读反馈与诊断结果
7. 根据反馈生成海报优化建议或优化版本

## 项目特点

- 将海报阅读行为纳入生成流程，而不只关注静态结果
- 使用语义区域而非纯坐标方式解释阅读反馈
- 同时结合注视估计与表情状态进行反馈判断
- 支持“分析 - 诊断 - 优化”闭环演示
- 适用于课程项目展示、交互原型验证与海报优化实验

## 注意事项

- 本项目依赖摄像头权限，运行时请允许浏览器访问摄像头
- `public/mediapipe/` 中的模型文件为运行所需资源，不建议删除
- `.env` 中可能包含密钥信息，请勿上传到公共仓库
- `dist/`、`node_modules/`、`logs/`、`outputs/` 等内容不属于源码主体
